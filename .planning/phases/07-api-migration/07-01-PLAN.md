    Create `src/lib/db-queries.ts` with these exported functions:

    **1. rowToSessionInfo(row) — internal helper:**
    Map DB row to SessionInfo. Parse JSON columns defensively:
    - `models`: `JSON.parse(row.models || '[]')` — these are raw model IDs from ingest (already display-name-mapped by parseSessionFile which calls getModelDisplayName). Check ingest.ts line 230: `JSON.stringify(s.models || [])` — s.models comes from reader.ts parseSessionFile which already maps to display names. So DB stores display names. Do NOT call getModelDisplayName again.
    - `toolsUsed`: `JSON.parse(row.tools_used || '{}')`
    - `compaction`: `JSON.parse(row.compaction || '{}')` — ensure it has all CompactionInfo fields with defaults: `{ compactions: 0, microcompactions: 0, totalTokensSaved: 0, compactionTimestamps: [], ...parsed }`

    **2. getDashboardStatsFromDb(): DashboardStats**
    Run these queries against getDb():
    - Totals: `SELECT COUNT(*) as totalSessions, SUM(message_count) as totalMessages, SUM(total_input_tokens + total_output_tokens + total_cache_read_tokens + total_cache_write_tokens) as totalTokens, SUM(estimated_cost) as estimatedCost, MIN(timestamp) as firstSessionDate FROM sessions`
    - Project count: `SELECT COUNT(*) as count FROM projects`
    - dailyActivity: `SELECT date, SUM(message_count) as messageCount, SUM(session_count) as sessionCount, SUM(tool_call_count) as toolCallCount FROM daily_activity GROUP BY date ORDER BY date ASC`
    - dailyModelTokens: `SELECT substr(timestamp, 1, 10) as date, model, SUM(total_input_tokens + total_output_tokens + total_cache_read_tokens + total_cache_write_tokens) as tokens FROM sessions WHERE model != '' AND model != 'unknown' GROUP BY date, model ORDER BY date ASC` — then transform into DailyModelTokens[] by grouping rows by date into `{ date, tokensByModel: { [model]: tokens } }`
    - modelUsage: `SELECT * FROM model_usage` — map each row to `ModelUsage & { estimatedCost: number }` where estimatedCost = cost_usd, and recalculate costUSD from `calculateCost(model, input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens)`. Set contextWindow=0, maxOutputTokens=0, webSearchRequests=web_search_requests||0.
    - hourCounts: `SELECT substr(timestamp, 12, 2) as hour, COUNT(*) as count FROM sessions GROUP BY hour` — SQLite substr is 1-indexed, so position 12 gives chars 12-13 of `2024-03-01T10:30:00Z` which is "10". Build Record<string, number>.
    - longestSession: `SELECT id, duration, message_count, timestamp FROM sessions ORDER BY duration DESC LIMIT 1` — map to LongestSession `{ sessionId: row.id, duration: row.duration, messageCount: row.message_count, timestamp: row.timestamp }`. If no rows, return `{ sessionId: '', duration: 0, messageCount: 0, timestamp: '' }`.
    - recentSessions: `SELECT * FROM sessions ORDER BY timestamp DESC LIMIT 10` — map each via rowToSessionInfo.
    - Return all fields assembled into DashboardStats.

    **3. getSessionsFromDb(limit = 50, offset = 0): SessionInfo[]**
    `SELECT * FROM sessions ORDER BY timestamp DESC LIMIT ? OFFSET ?` — map via rowToSessionInfo.

    **4. getProjectSessionsFromDb(projectId: string): SessionInfo[]**
    `SELECT * FROM sessions WHERE project_id = ? ORDER BY timestamp DESC` — map via rowToSessionInfo.

    **5. searchSessionsFromDb(query: string, limit = 50): SessionInfo[]**
    `SELECT * FROM sessions WHERE project_name LIKE ? OR git_branch LIKE ? OR cwd LIKE ? ORDER BY timestamp DESC LIMIT ?` — bind `%${query}%` three times + limit. Map via rowToSessionInfo.

    **6. getProjectsFromDb(): ProjectInfo[]**
    Two queries:
    a) `SELECT * FROM projects ORDER BY last_active DESC` — base project rows
    b) `SELECT DISTINCT project_id, model FROM sessions WHERE model != '' AND model != 'unknown' ORDER BY project_id, model` — collect models per project
    Build a Map<string, string[]> of project_id -> model display names. The `model` column in sessions stores raw IDs (e.g. `claude-opus-4-6`), but ingest writes `s.model` which is the raw ID from parseSessionFile (line 325: `model: models[0] || 'unknown'`). The `models` JSON column stores display names. So for the projects models array, read from sessions.model (raw) and apply getModelDisplayName().
    Map project rows to ProjectInfo[], using the models map for each project.

    **7. getSessionDetailFromDb(sessionId: string): Promise<SessionDetail | null>**
    a) `SELECT * FROM sessions WHERE id = ?` — if no row, return null
    b) Import and call `getSessionDetail(sessionId)` from reader.ts to get JSONL messages
    c) If getSessionDetail returns null (JSONL file missing), still return DB aggregates with empty messages: `{ ...rowToSessionInfo(row), messages: [] }`
    d) Otherwise: `{ ...rowToSessionInfo(row), messages: detail.messages }`
    This function is async because getSessionDetail reads JSONL.
