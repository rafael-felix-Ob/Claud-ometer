## Spacing Scale

Declared values (multiples of 4). Project uses Tailwind utility classes directly; these map to Tailwind's default scale.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, dot-to-text gap |
| sm | 8px | `gap-2`, `p-2` | Compact inline padding, badge padding |
| md | 16px | `gap-4`, `p-4` | Default element spacing, card grid gap |
| lg | 24px | `gap-6`, `p-6` | Section padding |
| xl | 32px | `gap-8`, `p-8` | Layout gaps |
| 2xl | 48px | `gap-12` | Major section breaks |
| 3xl | 64px | `gap-16` | Page-level spacing |

**Card-specific overrides (Claude's Discretion):**
- Session card internal padding: `p-4` (16px) — denser than the `p-5` (20px) used in StatCard, suits the richer data layout
- Card header bottom padding: `pb-3` (12px) — matches existing Card pattern in CLAUDE.md
- CardContent top padding: `pt-0` — matches existing Card pattern in CLAUDE.md
- Card grid gap: `gap-4` (16px) — locked in CONTEXT.md (`grid grid-cols-2 gap-4`)
- Status dot size: 8px × 8px (`h-2 w-2`) — visually distinct without overwhelming
- Dot-to-label gap: 6px (`gap-1.5`) — tight inline pairing

**Exceptions:** Touch targets (sidebar nav links) remain at minimum 44px tall per accessibility guidelines. Sidebar nav link padding `px-3 py-2` achieves this.

## Typography

Project uses Tailwind utility classes; no custom fontsize config in tailwind (Tailwind v4 via postcss). The following four sizes cover all roles needed on this page.

| Role | Size | Tailwind | Weight | Weight Class | Line Height | Usage |
|------|------|----------|--------|--------------|-------------|-------|
| Display | 20px | `text-xl` | Bold (700) | `font-bold` | 1.2 | Page title "Active Sessions (N)" |
| Heading | 14px | `text-sm` | Semibold (600) | `font-semibold` | 1.2 | Card title (project name), card section labels |
| Body | 14px | `text-sm` | Regular (400) | `font-normal` | 1.5 | Card body text, duration, token/cost display, message preview text |
| Label | 12px | `text-xs` | Medium (500) | `font-medium` | 1.4 | Stat labels, "last activity" timestamp, page subtitle, badge labels |
| Tiny | 10px | `text-[10px]` | Regular (400) | `font-normal` | 1.3 | Secondary metadata (path truncation, sidebar footer) |

**Weights in use: 2** — Regular (400) for body/tiny, Semibold (600) for headings. Bold (700) used only for the page title display role.

**Monospace usage:** Git branch name uses `font-mono` at 12px (`text-xs font-mono`) — locked in CONTEXT.md.

**Source:** CLAUDE.md text sizing conventions + CONTEXT.md card layout decisions.

## Color

The project uses CSS variables with a dark-default theme. Colors below are the dark theme values (the rendered defaults) from `globals.css`.

| Role | CSS Variable | Dark Value | Light Value | Usage |
|------|-------------|------------|-------------|-------|
| Dominant (60%) | `--background` | `#1C1C1E` | `#FAF9F6` | Page background |
| Secondary (30%) | `--card` | `#2C2C2E` | `#FFFFFF` | Session cards, sidebar, nav |
| Accent (10%) | `--primary` | `#E8956E` (Claude orange) | `#D4764E` | Specific elements only — see list below |
| Destructive | `--destructive` | `oklch(0.704 0.191 22.216)` | `oklch(0.577 0.245 27.325)` | No destructive actions in this phase |

**Accent (`--primary` / Claude orange) reserved for:**
- Sidebar active nav item background (`bg-primary/10`) and text (`text-primary`)
- Sidebar logo background square (`bg-primary`)
- StatCard icon container background (`bg-primary/10`) and icon color (`text-primary`)
- Status ring / focus ring (`--ring`)
- No other elements — status colors use semantic greens/ambers/grays (see Status Colors below)

**Status semantic colors (not accent — these are semantic signals):**

| Status | Dot Color | Border | Badge BG | Badge Text | Badge Border |
|--------|-----------|--------|----------|------------|--------------|
| Working | `bg-green-500 animate-pulse` | `border-l-green-500` | `bg-green-500/10` | `text-green-600` | `border-green-500/30` |
| Waiting | `bg-amber-500` (static) | `border-l-amber-500` | `bg-amber-500/10` | `text-amber-600` | `border-amber-500/30` |
| Idle | `bg-muted-foreground/40` | `border-l-border` | `bg-secondary` | `text-muted-foreground` | (none) |

**Left border spec:** `border-l-4` applied to Card `className`. Thickness is 4px — sufficient to clear `rounded-xl` corner clipping (avoids the invisible-border pitfall documented in RESEARCH.md).

**Idle card muting:** Idle card content uses `opacity-75` on the card body text — not on the entire card — to preserve scannability of the project name and status badge.

**Model badge colors:** Applied as inline `style={{ color: getModelColor(session.model) }}` on the badge span — NOT as Tailwind dynamic class. This avoids Tailwind v4 JIT purge of runtime-constructed color classes.

**Source:** CONTEXT.md status visual treatment + RESEARCH.md Pattern 6 + globals.css.

## Spacing & Layout Contract

### Page Layout

```
/active page (full width minus 60px sidebar)
├── Page header (mb-6)
│   ├── Title: "Active Sessions (N)"   text-xl font-bold tracking-tight
│   └── Subtitle: "Updated Xs ago"     text-sm text-muted-foreground
├── Imported data banner (if applicable, mb-4)
│   └── Amber warning bar              full width, p-3, rounded-lg
├── Summary stat row (mb-6)
│   └── 3–4 StatCards in grid-cols-3 or grid-cols-4 gap-3
└── Session card grid
    └── grid grid-cols-2 gap-4
        └── SessionCard (per active session, sorted: working → waiting → idle)
```

### Session Card Layout

```
Card (border-l-4, rounded-xl, border-border/50, shadow-sm, cursor-pointer)
├── CardHeader pb-3
│   ├── Row 1: Project name (text-sm font-semibold truncate) + Status badge (right)
│   └── Row 2: Duration prominent (text-xl font-bold) + "last activity" (text-xs text-muted-foreground)
├── CardContent pt-0
│   ├── Token/cost row: "1.2M tokens • $4.32"   text-sm text-muted-foreground
│   ├── Model badge row: colored badge           text-xs, inline style color
│   └── Git branch row: font-mono text-xs        GitBranch icon h-3 w-3
└── [Expanded section — conditionally rendered]
    ├── Separator
    ├── Last 4 messages (3 minimum, 5 maximum — default: 4)
    │   ├── User turn: bg-muted/50, rounded, p-2, text-xs
    │   └── Assistant turn: no background, p-2, text-xs
    └── "View full session →" link to /sessions/[id]   text-xs text-primary
```

### Page Header StatCards (summary row above grid)

3 cards, `grid grid-cols-3 gap-3`:
1. "Active Now" — count of sessions with status "working" or "waiting"
2. "Total Sessions" — total count of active sessions returned by the API
3. "Tokens (Recent)" — sum of all active sessions' token totals

## Interaction Contract

### Card Click — Expand/Collapse

- **Trigger:** Click anywhere on a session card
- **Behavior:** Toggle expanded state for that card. Only one card expanded at a time. Clicking an expanded card collapses it.
- **Implementation:** Local `expandedId: string | null` state at page level. Expanded section mounts an `<ExpandedCardDetail sessionId={id} />` child component that calls `useSessionDetail` internally.
- **Transition:** `max-height` CSS transition from `0` to `auto` using `tw-animate-css` `animate-in` / `animate-out` slide-in-from-top pattern, 150ms duration. Use `data-[state=open]` attribute pattern matching shadcn convention.
- **Collapsed default:** All cards start collapsed.

### Card Hover

- **Effect:** `hover:shadow-md hover:scale-[1.01]` — subtle lift
- **Cursor:** `cursor-pointer` on the card `div`
- **Transition:** `transition-all duration-150`

### Working Status Pulse

- **Element:** 8×8px dot (`h-2 w-2 rounded-full`)
- **Animation:** `animate-pulse` (Tailwind default, 2-second cycle)
- **No custom duration** — use Tailwind default. If a tighter cadence is later desired, add `@keyframes active-pulse` to `globals.css` rather than Tailwind arbitrary values.

### "View Full Session" Link

- **Placement:** Bottom of expanded section, right-aligned
- **Style:** `text-xs text-primary hover:text-primary/80 flex items-center gap-1`
- **Icon:** `ExternalLink h-3 w-3`
- **Navigation:** `<Link href={/sessions/${session.id}}>` — uses Next.js Link for client navigation
- **Does NOT close the expanded card** on click (navigates away)

### Last-Updated Indicator

- **Placement:** Page subtitle, inline with "Updated"
- **Update trigger:** `useEffect` fires whenever SWR `data` reference changes (`[sessions]` dependency)
- **Display:** `timeAgo()` from `src/lib/format.ts`
- **Loading state:** "Loading..." until first successful fetch

### Auto-Refresh

- **Interval:** 5000ms (`refreshInterval: 5000` in SWR hook)
- **No visual refresh spinner** — data updates silently; only the subtitle timestamp updates
- **Expanded card detail:** `<ExpandedCardDetail>` uses `useSessionDetail` WITHOUT polling (no `refreshInterval`) — static snapshot of the session at expand time

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Page title | `Active Sessions (N)` | N = integer count; if 0, still show "(0)" |
| Page subtitle — polling | `Updated just now` / `Updated 3s ago` | Uses `timeAgo()` output |
| Page subtitle — loading | `Loading...` | Before first successful fetch |
| Sidebar nav label | `Active` | Short — matches existing 1-word labels |
| Empty state heading | `No active sessions` | Concise, factual |
| Empty state body | `Claude Code isn't running in any projects right now. View your session history to review past activity.` | Includes link to /sessions |
| Empty state link | `View session history →` | Text link at end of body |
| Imported data banner | `Live monitoring unavailable` | Heading of banner |
| Imported data banner body | `You're viewing imported data. Switch to live mode to monitor active sessions.` | Body of banner |
| Working badge | `Working` | Matches SessionStatus value |
| Waiting badge | `Waiting` | Matches SessionStatus value |
| Idle badge | `Idle` | Matches SessionStatus value |
| Token/cost display | `{N} tokens • {cost}` | e.g. "1.2M tokens • $4.32"; bullet separator |
| Last activity label | `Active {timeAgo}` | e.g. "Active 2s ago" |
| Duration label | No prefix label — value only | e.g. "1h 23m" rendered as the prominent number |
| Expanded view label | `Recent messages` | Section heading above message list, `text-xs font-medium text-muted-foreground uppercase tracking-wide` |
| View full session link | `View full session` | Followed by ExternalLink icon |
| Loading spinner label | `Loading...` | Below spinner, `text-sm text-muted-foreground` |
| StatCard — active count | `Active Now` | Label |
| StatCard — total count | `Sessions` | Label |
| StatCard — tokens | `Tokens (Recent)` | Label; "(Recent)" qualifier per STATE.md open question on token semantics |
| Error state (API fail) | `Failed to load active sessions` | `text-sm text-muted-foreground` centered, no retry button (SWR retries automatically) |

**Destructive actions:** None in this phase. No confirmation dialogs needed.

**Truncation rules (Claude's Discretion):**
- Project name: `truncate` (single line, CSS ellipsis) — max width constrained by card header
- Project path: `truncate` at 40 characters, `text-xs text-muted-foreground`
- Git branch: `truncate` at card width — `font-mono text-xs` with `max-w-[160px]`
- Message preview in expanded view: clamp to 3 lines (`line-clamp-3`)
