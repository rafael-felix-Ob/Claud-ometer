  <name>readGsdProgress — GSD state reader</name>
  <files>src/lib/claude-data/gsd-progress.ts, src/__tests__/lib/gsd-progress.test.ts, src/lib/claude-data/types.ts</files>
  <behavior>
    - Tier 1 (non-GSD): `readGsdProgress('')` returns `null`
    - Tier 1 (non-GSD): `readGsdProgress('/path/no-planning')` returns `null` when `.planning/` dir does not exist
    - Tier 2 (GSD, unreadable): returns `{ isGsd: true, phaseName: null, ... }` when `.planning/` exists but `STATE.md` is absent
    - Tier 2 (GSD, unreadable): returns `{ isGsd: true, phaseName: null, ... }` when `STATE.md` exists but has malformed/empty content
    - Tier 3 (GSD, full): returns full `GsdProgress` object when valid `STATE.md` exists with frontmatter + prose
    - Tier 3: `phaseName` extracted from prose line `Phase: N of M (Name)` — e.g., `"Active Sessions Page"`
    - Tier 3: `phaseNumber` extracted as integer from same line — e.g., `2`
    - Tier 3: `phaseStatus` extracted from prose line `Status: ...` — e.g., `"Phase 2 complete — ready for Phase 3"`
    - Tier 3: `nextAction` derived as `/gsd:execute-phase {phaseNumber}` — e.g., `"/gsd:execute-phase 2"`
    - Tier 3: `totalPhases`, `completedPhases`, `percent` from frontmatter `progress:` block
    - Error safety: any exception during parse returns Tier 2 shape, never throws
  </behavior>
  <implementation>
    See 03-RESEARCH.md Pattern 2 (frontmatter parsing) and Pattern 3 (phase extraction). The function:

    1. Guard: if `!projectPath` return null
    2. Guard: if `.planning/` dir doesn't exist (existsSync) return null
    3. Guard: if `STATE.md` doesn't exist return GSD_UNREADABLE constant
    4. Read STATE.md with readFileSync (tiny file, sync is fine — document assumption)
    5. Parse YAML frontmatter with regex line scanner (no yaml library):
       - Match `^---\n(content)\n---` for frontmatter block
       - Parse top-level keys and one-level nested blocks (progress:)
       - Strip quotes from values, convert numeric strings to numbers
    6. Extract phase info from prose: `Phase: N of M (Name)` regex
    7. Extract status from prose: `Status: ...` regex
    8. Derive nextAction: `/gsd:execute-phase {phaseNumber}`
    9. Wrap entire parse in try/catch returning GSD_UNREADABLE on any error

    IMPORTANT anti-patterns from research:
    - Do NOT use frontmatter `status` field for phase status (it's milestone-level)
    - Do NOT import any yaml library
    - Do NOT throw — always return null or GSD_UNREADABLE
    - Always use `projectPath`, never `cwd`
  </implementation>
</feature>
